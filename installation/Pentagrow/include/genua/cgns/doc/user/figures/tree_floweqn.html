This diagram illustrates the layout of the relevant parts of the CGNS file for recording flow equation set information.

The Zone_t node is the same as in previous figures, with the following characteristics.

  • Name: Zone 1
    Label: Zone_t
    Data: VertexSize=(21,17,9), CellSize=(20,16,8), VertexSizeBoundary=(0,0,0)

Below the Zone_t node are multiple nodes, with the following two nodes shown in detail.

  • Name: ZoneType
    Label: ZoneType_t
    Data: Structured
  • Name: FlowEquationSet
    Label: FlowEquationSet_t
    Data: MT

Below the FlowEquationSet_t node are again multiple nodes, with the following three shown in detail.

  • Name: EquationDimension
    Label: "int"
    Data: 3
  • Name: GoverningEquations
    Label: GoverningEquations_t
    Data: NSTurbulent
  • Name: GasModel
    Label: GasModel_t
    Data: Ideal

Below the GoverningEquations_t nodes is a node defining the diffusion model used.

  • Name: DiffusionModel
    Label: "int[1+...+IndexDimension]"
    Data: (0,1,0,0,0,0)

Below the GasModel_t nodes is a child defining the specific heat ratio.

  • Name: SpecificHeatRatio
    Label: DataArray_t
    Data: 1.4

And finally, below the node defining the specific heat ratio is a child node defining the data class.

  • Name: DataClass
    Label: DataClass_t
    Data: NondimensionalParameter
